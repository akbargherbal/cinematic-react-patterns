PHASE 3: IMPLEMENT (30 min)

Lift State to Common Parent

‚ùå Before: State in Child

// Counter.jsx
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}

// Display.jsx
function Display() {
  // ‚ùå Can't access count!
  return <div>Count: ???</div>;
}

‚úÖ After: State in Parent

// App.jsx
function App() {
  const [count, setCount] = useState(0);
  
  return (
    <>
      <Counter count={count} setCount={setCount} />
      <Display count={count} />
    </>
  );
}

// Counter.jsx
function Counter({ count, setCount }) {
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}

// Display.jsx
function Display({ count }) {
  return <div>Count: {count}</div>;
}

‚¨áÔ∏è Data Flows DOWN via Props

App: count = 5
‚Üí Counter: props.count = 5
‚Üí Display: props.count = 5

‚¨ÜÔ∏è Updates Flow UP via Functions

Counter: onClick={() => setCount(6)}
‚Üí Calls function from App
‚Üí App state updates to 6
‚Üí Both children re-render with new value

üîë The Pattern:

  1. Move useState to common parent
  2. Pass state value down to components that read it
  3. Pass setter function down to components that update it
  4. Remove old local state from children

Context Setup Pattern

// ThemeContext.jsx
import { createContext, useContext, useState } from 'react';

const ThemeContext = createContext();

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  return useContext(ThemeContext);
}

// App.jsx
function App() {
  return (
    <ThemeProvider>
      <Layout />
    </ThemeProvider>
  );
}

// Any deep component
function Button() {
  const { theme, setTheme } = useTheme();
  return <button className={theme}>Toggle</button>;
}

  • ‚Ä¢ You're new to React (master props first!)

üéØ Common Mistakes to Avoid

‚ùå Anti-Patterns:

  • ‚Ä¢ Lifting state TOO high (makes App bloated)
  • ‚Ä¢ Duplicating state in multiple places
  • ‚Ä¢ Prop drilling when Context makes sense
  • ‚Ä¢ Using Context for everything

‚úÖ Best Practices:

  • ‚Ä¢ Start local, lift only when needed
  • ‚Ä¢ Keep state as low as possible
  • ‚Ä¢ One source of truth per state
  • ‚Ä¢ Name state clearly: count, not data

Events: Bubble UP like air (child ‚Üí parent)
Source of Truth: Only ONE place holds the real data
Components: Dumb receivers, smart parent decides
